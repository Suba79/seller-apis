# Интеграция с маркетплейсами

## Что делают скрипты
Скрипты следят за актуальностью информации о часах Casio на двух маркетплейсах:

### Для Ozon (seller.py)
- Автоматически скачивают свежий прайс-лист с остатками от поставщика (timeworld.ru)
- Сверяют, какие модели уже есть в магазине на Ozon
- Обновляют цены — они берутся прямо из прайс-листа поставщика
- Обновляют остатки — если на складе поставщика осталось мало часов, на Ozon это тоже отобразится

### Для Яндекс.Маркета (market.py)
Скрипт работает похожим образом, но с учётом особенностей двух схем работы:
- **FBS** (продавец хранит товары на своём складе, но использует доставку Маркета)
- **DBS** (продавец сам хранит и доставляет товары)

Для каждой схемы отдельно обновляются цены и остатки.

## Откуда берутся данные
Всё берётся из одного источника — файла остатков с сайта timeworld.ru. Там хранится актуальная информация о наличии и ценах на часы Casio.

## Важные особенности
- **Цены** не придумываются продавцом — они автоматически подставляются из прайса поставщика
- **Остатки** обрабатываются по логике: если на складе больше 10 часов — показываем запас 100, если остался 1 час — считаем, что товара нет в наличии
- Все обновления происходят автоматически при запуске скриптов

## Технические детали (для информации)
Для работы нужны токены доступа к API Ozon и Яндекс.Маркета, а также идентификаторы кампаний и складов. Они настраиваются через переменные окружения.
