# Интеграция с маркетплейсами

Скрипты для автоматического обновления цен и остатков часов Casio на Ozon и Яндекс.Маркете.

## Что делают скрипты

Скрипты следят за актуальностью информации о часах Casio на двух маркетплейсах:

### Для Ozon (seller.py)
- Автоматически скачивают свежий прайс-лист с остатками от поставщика (timeworld.ru)
- Сверяют, какие модели уже есть в магазине на Ozon
- Обновляют цены — они берутся прямо из прайс-листа поставщика
- Обновляют остатки — если на складе поставщика осталось мало часов, на Ozon это тоже отобразится

### Для Яндекс.Маркета (market.py)
Скрипт работает с API Яндекс.Маркета и поддерживает две модели работы:

**FBS (Fulfillment by Seller)** — товары хранятся на складе продавца, но доставкой занимается Маркет:
- Скрипт обновляет остатки на складе продавца
- Маркет забирает товары и доставляет покупателям

**DBS (Delivery by Seller)** — продавец самостоятельно хранит и доставляет товары:
- Скрипт обновляет остатки и цены
- Продавец сам организует доставку до покупателя

Для каждой модели используются свои идентификаторы кампании и склада — всё обновляется отдельно.

## Откуда берутся данные

Всё берётся из одного источника — файла остатков с сайта timeworld.ru. Там хранится актуальная информация о наличии и ценах на часы Casio. Файл скачивается автоматически при каждом запуске скриптов.

## Важные особенности

- **Цены** не придумываются продавцом — они автоматически подставляются из прайса поставщика
- **Остатки** обрабатываются по особой логике:
  - Если на складе больше 10 часов — в маркетплейсы передаётся запас 100
  - Если остался 1 час — считается, что товара нет в наличии (передаётся 0)
  - В остальных случаях передаётся фактическое количество
- **Артикулы (SKU)** — используются внутренние коды товаров из прайса поставщика
- Все обновления происходят автоматически при запуске скриптов

## Технические детали (для разработчиков)

Для работы нужны токены доступа к API Ozon и Яндекс.Маркета, а также идентификаторы кампаний и складов. Они настраиваются через переменные окружения:

**Для Ozon:**
- `SELLER_TOKEN` — токен продавца
- `CLIENT_ID` — идентификатор клиента

**Для Яндекс.Маркета:**
- `MARKET_TOKEN` — токен для доступа к API
- `FBS_ID` и `DBS_ID` — идентификаторы кампаний
- `WAREHOUSE_FBS_ID` и `WAREHOUSE_DBS_ID` — идентификаторы складов
